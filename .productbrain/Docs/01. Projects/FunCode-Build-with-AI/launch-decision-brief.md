## 1. Domain name (**DEC-7**)

**Settled:** FunCode launches on a **new domain**, not `community.productbrain.io`. Build with AI is a community brand to own; Product Brain remains a governance tool in the landscape (**DEC-7**, **LAND-6**). The specific domain string is still **TBD** — below the Waterline until Randy signs off (**BR-3**, **QUE-1/2/3**).

**Still open:** Own domain vs. a Product Brain surface (**QUE-3**, archived **TEN-8**) — downstream of community-led vs product-led (**QUE-1**). Community-led → own the audience asset on a standalone URL; product-led → community as a surface under `productbrain.io`. Lock domain after lead motion is chosen.

### Candidate direction A — `funcode.io` (umbrella brand)

| Pros | Cons |
|------|------|
| Matches working project/repo identity and Randy's "code is fun/play" thesis (**QUE-2**) | Does not carry "build with AI" in the URL — SEO for the promise line (**GLO-3**) needs H1/title support (**QUE-2** watch-out) |
| Supports proposed hierarchy: FunCode = umbrella belief; The Play Room = community place; Build with AI = tagline (**QUE-2**, **TEN-7**) | Less instantly descriptive for cold traffic who search "build with AI" |
| Aligns with **DEC-7** brand separation from Product Brain | Availability, typos, and `.io` perception for non-dev ICP need checking |
| Owns the audience asset if community-led wins (**QUE-3**) | Hard-to-reverse public commitment — below Waterline (**BR-3**) |

### Candidate direction B — `buildwithai.*` (promise / SEO line)

| Pros | Cons |
|------|------|
| Direct match to **GLO-3** / **STR-1** promise — strong search intent | Generic; crowded namespace; may read as course-mill or hype site (**STD-4** violation risk) |
| Keeps "Build with AI" as the front door (**DEC-7**: community brand to own) | Collapses the three-name hierarchy (**QUE-2**) — nowhere obvious for "FunCode" or "The Play Room" |
| Good for community-led motion where the movement name *is* the URL | Product Brain sibling relationship less clear (**LAND-6**, **INS-11**) |
| | Likely need redirects if naming settles on FunCode umbrella later |

### Candidate direction C — `playroom.*` (signature idea as URL)

| Pros | Cons |
|------|------|
| The URL teaches **GLO-2** on arrival — PR = play, not merge-to-prod (**QUE-2**, **INS-5**) | **GLO-2** collision: "Play Room" currently means the PR reframe, not the community (**QUE-2**, **TEN-7**) — needs disambiguation copy |
| Memorable, on-brand for playful-yet-humble voice (**DEC-12**, **STD-4**) | B2B / cohort buyer may read "playroom" as unserious (**QUE-2** watch-out) |
| Pairs with content pillar Play Rooms (**STR-3**) | Weaker SEO for "build with AI" unless compensated in title/H1 (**QUE-2**) |
| Distinct from Product Brain (**DEC-7**) | Childcare / gaming namespace noise; availability uncertain |

**Pre-decision checks (all directions):** WHOIS + trademark skim · redirect plan from any interim URL · explicit Randy sign-off before DNS commit (**BR-3**).

## 3. Community naming (**QUE-2** / **TEN-7**)

**Open question:** Should the community be named "The Play Room"? Is "FunCode" placeholder or real umbrella brand? (**QUE-2**)

**Context:** Randy's thesis — code is fun/play — is FunCode's meaning, so FunCode + The Play Room share one metaphor, not competing names (**QUE-2**, **TEN-7**).

### Option A — FunCode as front door

| Pros | Cons |
|------|------|
| Matches repo, workspace, and current working identity | "FunCode" reads as placeholder to some founders (**QUE-2**) |
| Encodes play/fun thesis directly | Does not teach **GLO-2** PR reframe from the name alone |
| Clean umbrella if hierarchy is FunCode → community → Build with AI tagline (**QUE-2**) | Domain/naming collision with **DEC-7** TBD |

### Option B — "The Play Room" as community name (FunCode umbrella)

| Pros | Cons |
|------|------|
| Name teaches PR = play (**QUE-2**, **GLO-2**, **INS-5**) | **GLO-2** collision: singular "The Play Room" = place vs "a Play Room" = individual PR (**TEN-7**) |
| Proposed hierarchy: FunCode = belief; The Play Room = place; Build with AI = SEO line (**QUE-2**) | B2B/cohort "playroom" perception (**QUE-2** watch-out) |
| Onboarding already situates work in Play Room context (**DEC-20**) | Requires consistent copy discipline across all surfaces (**STD-4**) |

### Option C — "Build with AI" as community name (demote to tagline in hierarchy)

| Pros | Cons |
|------|------|
| Strongest SEO / cold-traffic clarity (**GLO-3**, **STR-1**) | Three-name budget blown — FunCode and Play Room both demoted (**QUE-2**) |
| Promise-forward for **AUD-1** agent-curious product people | Reads generic; harder to own as a durable brand (**DEC-7**) |

**Disambiguation pattern (if Option B wins):** "The Play Room" (singular) = community; "a Play Room / Play Rooms" = individual PRs (**QUE-2**, **TEN-7**).

**Do not decide here.** Naming is below Waterline when published broadly (**BR-3**, **QUE-1/2/3**).

## 4. Testimonials (**TEN-5**)

**Problem:** Landing ships with six **placeholder** quotes in FunCode voice — illustrative scaffolding, **not** real endorsements (**TEN-5**). Fabricated social proof on a live site breaches **INS-9** (prove value via real artifacts) and **BR-3** (no false/fabricated public claims).

### Process to collect real quotes (before public launch)

| Step | Action | Guardrail |
|------|--------|-----------|
| 1. **Identify sources** | Early builders from beta cohort, fork-this-repo adopters (**DEC-8**), Play Room contributors with shipped artifacts | Real people only — no invented names |
| 2. **Ask with context** | Short ask: what changed after prototyping with agents / Play Rooms? Link to **GLO-2**, **FLO-1** outcome | Peer voice, not marketing polish (**STD-4**) |
| 3. **Consent in writing** | Explicit OK to publish: full name + role (or agreed pseudonym + role), quote text, optional headshot | **BR-3** — names/quotes without consent = below Waterline |
| 4. **Capture on Chain** | `pb capture` testimonial as **INS** with `--source-ref` (DM/email/thread URL) and link to member if appropriate | **PAT-6** traceability |
| 5. **Replace or remove** | Swap placeholder array in `web/src/routes/+page.svelte` with consented quotes **or** remove section and show real Play Room outputs instead (**INS-9**, **INS-10**) | No launch with placeholders (**TEN-5**) |
| 6. **No fabricated metrics** | Headlines stay non-numeric until real counts exist (**TEN-5**) | No invented "500+ builders" |

### Alternative to testimonials section

If consented quotes aren't ready at launch: replace marquee with **real Play Room outputs** — Chain artifacts, merged PR links, demo clips (**INS-9**, **INS-10**). Better empty or real than fake.
